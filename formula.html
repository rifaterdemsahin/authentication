Fly.io Integration Formula

How code goes from your laptop to a live URL

Master Formula

git push → GitHub Actions → Doppler → flyctl deploy → Live on Fly.io

Step-by-Step Breakdown

1

Write Code Locally

Edit app.py, index.html, etc. on your machine. Flask runs on port 8080.

2

git push → main

Pushing to the main branch triggers the GitHub Actions workflow at .github/workflows/fly-deploy.yml.

3

GitHub Actions runs

The runner checks out your code, installs flyctl, and reads FLY_API_TOKEN from GitHub Secrets.

4

Docker image built

Fly.io builds the Dockerfile remotely using Depot. Python deps are installed from requirements.txt.

5

Machine deployed

Fly.io replaces the old machine with the new image — zero downtime via two machines in lhr (London).

6

Live URL ready

App is served at authentication-white-wildflower-5793.fly.dev over HTTPS automatically.

Where Secrets Live

Secret Stored In Used By Purpose
FLY_API_TOKEN GitHub Secrets GitHub Actions Authenticates flyctl to deploy your app
FLY_API_TOKEN Doppler Local / other pipelines Same token, managed centrally in Doppler
JWT_SECRET_KEY Fly.io Secrets Flask app at runtime Signs and verifies JWT tokens for users
DOPPLER_TOKEN GitHub Secrets GitHub Actions Lets Actions pull secrets from Doppler
